Case Study: Brite Weather, a Mobile Weather Application
Use Case Overview

To create a Weather application that is simple to navigate, easy to understand, and reliable to use. The goal of this project is to identify the applications users' needs. But to also understand any pain points they might have when interacting with the application. I've employed several UX methodologies for this case study. I've created a user persona to understand who would be using the product and why. I've created a storyboard with a specific user scenario to understand the users' mindset and why there is a need for the Brite Weather app. I've sketched out ideas to refine the design. I've conducted a usability test to understand what is important to the user and use that feedback to develop the final deliverable, a prototype.
